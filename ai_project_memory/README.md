# AI Project Memory - Panduan Kontinuitas Proyek

Folder ini berisi memori kolektif proyek yang dirancang khusus agar **AI Coding Assistant** dan **Developer Manusia** dapat bekerja secara selaras, konsisten, dan aman.

## Mengapa folder ini ada?
Untuk mencegah asisten AI melakukan "halusinasi" arsitektur atau merusak modul yang sudah stabil saat berpindah sesi, berpindah komputer, atau saat editor ditutup.

## Cara Menggunakan (Untuk AI):
1. **BACA PERTAMA**: Selalu baca `PROMPT_START.md` sebelum memulai pekerjaan baru.
2. **KONTEKS**: Gunakan `AI_CONTEXT.md` dan `PROJECT_ARCHITECTURE.md` untuk memahami cara kerja sistem.
3. **ATURAN**: Patuhi `AI_RULES.md` dan hormati file yang ada di `FIXED_MODULES.md`.
4. **UPDATE**: Selalu update `CURRENT_PROGRESS.md` dan `CHANGELOG.md` setiap kali ada perubahan signifikan.

## Daftar File:
- **`AI_CONTEXT.md`**: Visi besar dan stack teknologi.
- **`AI_RULES.md`**: Batasan dan standar coding.
- **`PROJECT_ARCHITECTURE.md`**: Diagram alur dan struktur sistem.
- **`CURRENT_PROGRESS.md`**: Status fitur dan session terakhir.
- **`FIXED_MODULES.md`**: Daftar file yang dilarang/sensitif untuk diubah.
- **`CHANGELOG.md`**: Histori perubahan teknis.
- **`TODO.md`**: Rencana kerja berdasarkan prioritas.
- **`PROMPT_START.md`**: Instruksi wajib bagi AI baru.
- **`SYSTEM_STATE.json`**: Snapshot status sistem (Machine-readable).

---
